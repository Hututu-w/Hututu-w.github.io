Oreder-Revealing Encryption

1 Introduction

1.1 Extending ORE: The Left/Right Framework

  1. left/right framework: the "left" encryption function and "right" encryption function takes a message and the secret key and outputs either a "left" or a "right" ciphertext respectively.

  2. comparison function: takes a left ciphertext and a right ciphertext, and outputs the comparison relation between the two underlying messages (encrypted by the left and right ciphertexts).

1.2 Our Contributions

An efficient small-domain ORE. with the best-possible semantic security that only relies on pseudorandom functions (PRFs). "small" is due to the ciphertext length grows linearly in the size of the plaintext space.

Domain extension for ORE. apply domain-extension technique to obtain an ORE scheme whose leakage profile is better than Chenette et al.

Encrypted range queries. The server sores the "right" ciphertext, the client provides the "left" ciphertexts corresponding to its range.

New lower bounds for OPE. no efficient OPE scheme can satisfy best-possible security.

2 Preliminaries

2.1 Order-Revealing Encryption

three algorithms: ORE.Setup, ORE.Encrypt, ORE.Compare defined over a well-ordered domain D:
- \(ORE.Setup(1^\lambda)\to sk\): on input a security parameter \(\lambda\), outputs a secret key \(sk\).
- \(ORE.Encrypt(sk,m)\to ct\): on input a secret key \(sk\) and a message \(m\in\)D, outputs a ciphertext \(ct\).
- \(ORE.Compare(ct_1,ct_2)\to b\): on input two ciphertext \(ct_1,ct_2\), outputs a bit \(b\in\{0,1\}\).

3 ORE for small-domain

Construcion. the message space \([N]\), secure PRF \(F:\{0,1\}^\lambda×\{0,1\}^\lambda\to\{0,1\}^\lambda\), hash function \(H:\{0,1\}^\lambda×\{0,1\}^\lambda\to\mathbb{Z}_3\), the comparison function \(CMP\), our ORE scheme is defined as follow:
- \(ORE.Setup(1^\lambda)\): samples a PRF key \(k\gets\{0,1\}^\lambda\) and a uniformly random permutation \(\pi:[N]\to[N]\), \(sk := (k,\pi)\).
- \(ORE.Encrypt_L(sk,x)\): \(sk := (k,\pi), ct_L=(F(k,\pi(x)), \pi(x))\).
- \(ORE.Encrypt_R(sk,y)\): \(sk := (k,\pi)\), samples a random nonce \(r\gets\{0,1\}^\lambda\), for each \(i\in[N], v_i=CMP(\pi^{-1}(i),y)+H(F(k,i),r)(mod\ 3), ct_R=(r,v_1,v_2,...,v_N)\).
- \(ORE.Compare(ct_L,ct_R)\): \(ct_L=(k',h), ct_R=(r,v_1,v_2,...,v_N)\), outputs \(v_h-H(k',r)(mod\ 3)\).
